# Local stack

The agicash workspace needs three local services running before any
end-to-end work: the OpenSecret enclave on `:3999`, an OpenSecret-backing
Postgres on `:5432`, and the Supabase local stack on `:54321`. The flake's
default shell points the workspace at these URLs by default.

## OpenSecret enclave

OpenSecret runs from a separate checkout. The first-time setup steps live in
the `opensecret` repo's README; the day-to-day commands are:

```sh
# postgres backend (nix-native service; one-time setup outside this repo)
# - listens on :5432
# - database name + JWT secret seeded by the opensecret repo's bootstrap

# enclave server
cd ~/opensecret
cargo run                        # listens on :3999
```

The enclave reads its JWT signing secret from the postgres backend at boot;
it must match `SUPABASE_JWT_SECRET` so the JWT chain (OpenSecret session →
third-party token → Supabase) verifies cleanly.

## Supabase

```sh
bunx supabase start              # postgres :54322 (internal), api :54321 (https)
bunx supabase status             # prints anon + service-role keys to paste into .env
bunx supabase stop
```

Supabase config: `supabase/config.toml`. Local TLS uses the mkcert-issued
cert at `certs/localhost-cert.pem`, generated by the flake's `generate-ssl-cert`
helper on first shell entry.

## mkcert trust on mobile devices

iOS / Android on the same Wi-Fi network need the mkcert root CA installed
to trust the local cert. Find it via:

```sh
mkcert -CAROOT
# typically ~/Library/Application Support/mkcert/rootCA.pem
```

AirDrop / email the `rootCA.pem` to the device, install via **Settings →
General → VPN & Device Management**, then enable trust in **Settings →
General → About → Certificate Trust Settings**.

## Sanity check

From inside the agicash repo, after `direnv allow`:

```sh
acli auth guest
acli account list
```

If `auth guest` fails with a connection refused, the OpenSecret enclave
isn't up. If `account list` fails with a TLS or 401 error, Supabase isn't
up or `.env` is missing the regenerated anon key.
